# Investigation Summary

This case study follows a structured SOC investigation process.

## Investigation Workflow

1. Initial observations
2. Hypothesis development
3. Host analysis
4. SIEM validation
5. Timeline reconstruction
6. IOC identification
7. MITRE ATT&CK mapping
8. Response recommendations

## Objective

The goal was to determine whether malicious activity had occurred, identify attacker techniques, and understand how persistence and privileged access were established.

## Outcome

The investigation identified multiple indicators of compromise affecting both Windows and Linux systems, including persistence mechanisms, privilege escalation activity, suspicious PowerShell execution, and unauthorized remote access tooling.
